คำตอบ

ยังไม่มี framework ไหนเป็น de facto standard — กลางปี 2026 สนามนี้กำลัง แตกออก (fragmenting) มากกว่ารวมกัน สัญญาณที่แรงที่สุดคือ Spec-Driven Development กลายเป็น "ชั้นฐานร่วม" ที่เป็นจุดเริ่มของทุก vendor framework — ถ้าจะบอกว่าอะไรชนะ mindshare คือ paradigm นี้ ไม่ใช่ framework หรือ standards body ตัวใดตัวหนึ่ง

The contenders

ผู้ท้าชิง 5 framework

แต่ละตัวมาจากคนละที่ — งานวิชาการ, vendor ใหญ่, และ pattern ทั่วไป — โดยมี spec-driven เป็นชั้นที่ทุกตัวใช้ร่วมกัน

V-Bounce

Academic · arXiv
  • ดัดแปลง V-model — AI เป็น implementation engine
  • คนเปลี่ยนบทบาทเป็น validator/verifier
  • บนกระดาษเท่านั้น — ไม่พบหลักฐาน adoption

AWS AI-DLC

Vendor · open-sourced
  • 3 เฟส: Inception → Construction → Operations
  • ปล่อยเป็น Amazon Q Rules + Kiro Steering Files
  • tool-agnostic (Q, Kiro, Cursor, Cline, Claude Code, Copilot)

MS / GitHub pipeline

Vendor · concrete
  • 5 สเตจ: Spec Kit → Copilot agent → Code Quality → Actions → SRE Agent
  • SRE Agent ปิด loop กลับผ่าน GitHub issues
  • รูปธรรมที่สุด มี reference จริง

Phase-mapping

Generic pattern
  • map AI tool เข้าแต่ละเฟส SDLC (7–8 เฟส)
  • เป็น pattern ทั่วไปในบทความ ไม่ใช่ framework เดียว
  • ยืดหยุ่นแต่ไม่มีโครงร่วม

Spec-Driven Development — ชั้นฐานร่วม (substrate) ⭐

~120,000 ★ · winning mindshare
  • ไม่ใช่ framework เต็ม แต่เป็น entry point ที่ทุกเจ้าเอาไปต่อหัวขบวน — Microsoft: "Spec Kit really is the entry point that triggers the flow"
  • GitHub Spec Kit = สัญญาณ adoption แรงสุด: ~119,500 ★ / ~10,600 forks, tool-agnostic กับ agent 30+ ตัว (official 42)
  • workflow: specify → plan → tasks → implement (ขยายเป็น 7–8 คำสั่ง รวม constitution, clarify, analyze, converge)

Caveats

  • แหล่ง primary ส่วนใหญ่เป็น vendor blog — analyst อิสระจริงมีแค่ Thoughtworks
  • ไม่มี Gartner MQ / Forrester Wave ครอบคลุม category นี้
  • V-Bounce เป็น proposal บนกระดาษ ไม่พบ adoption
  • ข้อสรุป "winning mindshare" อิงจาก star count + การที่ SDD โผล่เป็น entry point ซ้ำ ๆ
  • ข้อมูลเปลี่ยนเร็ว — star, ring, tool list เปลี่ยนรายเดือน

Claims ที่ถูกตีตก

  • "AI-DLC วางตัวเป็น replacement ของ SDLC เดิม" — ตีตก 1-2
  • "ไม่มี Gartner/Forrester coverage AIDLC เป็น construct เดียว" — ตีตก 0-3

คำถามที่ยังเปิด

  • จะมี standards body เป็นกลางเกิดขึ้น หรือ de facto จะมาจาก vendor เท่านั้น?
  • สองค่าย (minimal vs detailed) จะรวมกันไหม?
  • adoption จริงระดับ production ลึกแค่ไหน?
